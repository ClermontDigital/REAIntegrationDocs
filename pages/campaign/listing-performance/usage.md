# Listing Performance Usage

## Get Listing Performance Reports for Agency

**Get multiple listing performance reports by agency ID**

Returns a collection of listing performance reports for realestate.com.au property advertising campaigns belonging to an agent. The response includes reports for realestate.com.au listings that match the supplied query parameters. Each listing performance report includes details about the realestate.com.au listing and a list of portal metrics for the report period. For more information about other listing types, such as commercial properties, please [Contact Us](https://help.realestate.com.au/hc/en-us).

Please refer to response payload below for more details.

Each portal metrics object contains metrics at two different aggregation levels:

-   all (totals for the whole reporting period)
-   daily (totals for calendar dates for the whole reporting period)

Supports the following query parameters:

-   agency\_id

\*_Not available for Commercial listings performance._

### URL

Terminal window

```
GET https://api.realestate.com.au/campaign/v1/listing-performance
```

### Response Payload

#### Content Type

application/hal+json

#### Response Properties

The response payload uses the [HAL hypermedia content type](https://en.wikipedia.org/wiki/Hypertext_Application_Language) for embedding resources, and to support pagination of results and API links for individual resources. For more details on pagination, refer to the [Pagination](#pagination) section below.

The listing-performance resources are returned in the `items` array within the `_embedded` response property. The `_links` property for each embedded resource contains a `self` URL, a direct link for requesting that single item.

The `_links` property at the root level response contains a `self` URL for the actual request, and optionally `prev` and/or `next` URLs, which navigate to the previous or next page of results if they exist.

#### Pagination

The listing-performance items (embedded resources) are paginated using the [HAL hypermedia content type](https://en.wikipedia.org/wiki/Hypertext_Application_Language) standard. The API returns a maximum of 20 records at a time, and is ordered by the `id` property of the items. If the query result contains more than 20 items, the first page of 20 items will be returned, and the `_links` property will contain a `next` encoded hyperlink to navigate to the next page of results.

Upon navigation to the next page, the result will then contain a `prev` encoded hyperlink to navigate to the previous page of results (in this case the first page). If there are yet more results pages, the result will again contain a `next` hyperlink to navigate to the third page of results, and so on. The first (or only) page of results will not contain a `prev` hyperlink, and the last (or only) page of results will not contain a `next` hyperlink.

For example:

```
{  "_embedded": {    "items": [      // Listing performance reports for requested agency ID    ]  },  "_links": {    "self": {      "href": "https://api.realestate.com.au/campaign/v1/listing-performance?agency_id=ABCDEF&after=cBzYx7N0jpop9ItntUNppnYxLzMv"    },    "prev": {      "href": "https://api.realestate.com.au/campaign/v1/listing-performance?agency_id=ABCDEF&before=Sqf_dyQ6BGvjFZK38NFAPnYxLzMv"    },    "next": {      "href": "https://api.realestate.com.au/campaign/v1/listing-performance?agency_id=ABCDEF&after=qwck09xRNzr_4UcsJFh8wHYxLzIv"    }  }}
```

## Get Listing Performance Report by Listing ID

**Get single listing performance report by listing ID**

The listing-performance by listing-id endpoint returns a single listing performance report for a realestate.com.au or realcommercial.com.au property advertising campaign. The performance report is available for both active and inactive listing. Please see the [Campaign API FAQs](../overview.md#faq) for date restrictions. The response is a report for the listing that matches the supplied listing ID. The listing performance report includes details about the listing, and a list of portal metrics for the report period.

Each portal metrics object contains metrics at two different aggregation levels:

-   all (totals for the whole reporting period)
-   daily (totals for calendar dates for the whole reporting period)

### URL

```
GET https://api.realestate.com.au/campaign/v1/listing-performance/:listing_id
```

### Path Parameters

Name

Type

Required

Description

Example

listing\_id

integer

Required

The unique listing ID for the listing performance report

100012345

### Response Payload

#### Content Type

application/hal+json

#### Response Properties

The response payload uses the [HAL hypermedia content type](https://en.wikipedia.org/wiki/Hypertext_Application_Language) for embedding resources and API links. Single listing-performance resources are returned as a root level response object, with the `_links` property containing the `self` URL of the listing-performance resource, which in this case is the same as the requested URL.

The listing-performance resource has the same properties as the [listing-performance item payload](#listing-performance-item).

### Example Request

Terminal window

```
$ curl -H "Authorization: Bearer 41cd46e9-3a68-42e0-922e-c5c885366788" https://api.realestate.com.au/campaign/v1/listing-performance/100012345
```

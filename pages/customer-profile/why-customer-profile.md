# Why Customer Profile API

Agent profile data is one of REA’s most visible assets: it powers every “Find an Agent” experience, listing card, and agency profile page on realestate.com.au. When that data is stale, incomplete, or inconsistent, it reflects on both the agency and REA.

The Customer Profile API gives integrating partners a direct, programmatic channel to keep profile data in sync between their systems and REA. No more double-handling. No more manual re-entry.

## The problem it solves

Today, many agencies maintain agent profile data in their own platform and then separately re-enter or update the same information in REA tools. This creates:

-   **Double-handling overhead:** agency staff spend time on admin that could be automated
-   **Data lag:** profiles on realestate.com.au fall behind what the agency already knows
-   **Inconsistency risk:** small differences between systems compound over time into trust and accuracy issues
-   **Offboarding gaps:** when an agent leaves, delays in deactivating their REA profile can affect listings, branding, and data accuracy

## What it enables

The Customer Profile API lets your system become the source of truth for agent profile data. When an agent joins an agency, gets promoted, or moves on, your platform can push that change to REA automatically.

## What success looks like

For your integration, success means:

-   **Less admin work** for agency users: profile updates happen without leaving your platform
-   **Higher profile completeness:** required fields are populated consistently because your system is authoritative
-   **Faster updates:** changes are reflected on realestate.com.au without delay
-   **Clean offboarding:** agent departures are handled in a single step, with no dangling associations

## Who it’s for

The Customer Profile API is available to approved integration partners.

This includes:

-   **CRM providers** building integrations for real estate agencies
-   **Agency platforms** managing profiles across multiple offices or brands

Note

The API is currently in a controlled pilot. To request access, contact your REA account representative.
